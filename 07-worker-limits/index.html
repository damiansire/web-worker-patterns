⚠️ Límites de Workers

Ejemplo 07: Cantidad máxima y gestión de recursos

💡 ¿Qué demuestra este ejemplo? Los navegadores tienen límites en la cantidad de workers que pueden ejecutarse simultáneamente. Este ejemplo te permite crear múltiples workers y ver qué sucede cuando alcanzas esos límites.
⚠️ Límites típicos por navegador:
  • Chrome/Edge: ~20-50 workers (depende de núcleos CPU)
  • Firefox: ~20-30 workers
  • Safari: ~10-20 workers

El límite real depende de: memoria disponible, núcleos CPU, y configuración del navegador.

💡 Información Importante

¿Qué sucede cuando se excede el límite?

  • El navegador puede rechazar la creación de nuevos workers
  • Puede haber excepciones o errores silenciosos
  • El rendimiento general de la página puede degradarse
  • Otros procesos del navegador pueden verse afectados

Mejores Prácticas:

  • Pool de Workers: Reutiliza workers en lugar de crear nuevos
  • Límite Razonable: Usa 4-8 workers para la mayoría de aplicaciones
  • Monitoreo: Implementa contadores y limpieza automática
  • Terminación: Siempre termina workers cuando no los necesites

Detectar límites programáticamente:

const cpuCores = navigator.hardwareConcurrency || 4;
const maxWorkers = cpuCores * 2; // Regla general
