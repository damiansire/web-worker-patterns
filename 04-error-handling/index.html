⚠️ Manejo de Errores en Workers

Ejemplo 04: Capturando y manejando errores

💡 ¿Qué demuestra este ejemplo? Los errores que ocurren dentro de un Web Worker deben ser manejados adecuadamente. Este ejemplo muestra diferentes tipos de errores y cómo capturarlos usando el evento onerror.

❌ 1. Error de Referencia

Intentar usar una función o variable que no existe.

🔢 2. Error de Tipo

Intentar realizar una operación con un tipo de dato incorrecto.

➗ 3. Error de División

Operaciones matemáticas inválidas o problemas aritméticos.

🎯 4. Error Personalizado

Lanzar un error custom con un mensaje específico.

✅ 5. Operación Exitosa

Enviar una operación que se completa sin errores.
